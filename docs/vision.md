# Vision

## Why are we building Project Memory?

Software teams are very good at preserving code.

Git stores every commit.
Jira stores every ticket.
Slack stores every conversation.
Meeting tools store recordings.
Documentation lives in Notion or Confluence.

Yet, one thing is consistently lost over time:

**The reasoning behind engineering decisions.**

Months later, developers can usually answer:

- What changed?
- Who changed it?
- When was it changed?

But they struggle to answer:

- Why was this decision made?
- What alternatives were considered?
- What problem was the team trying to solve?
- Why was one solution chosen over another?
- What happened after the decision?

As projects grow, this context becomes fragmented across multiple tools and eventually disappears.

Project Memory exists to preserve and reconstruct that context.

---

# Vision Statement

Build an engineering memory system that helps teams understand the reasoning behind technical decisions instead of simply retrieving documents.

---

# Who is this for?

Initially, Project Memory is designed for software engineering teams.

Primary users include:

- Backend Developers
- Frontend Developers
- Full Stack Developers
- Tech Leads
- Engineering Managers
- New Team Members

---

# What problem are we solving?

We're not solving document search.

We're solving engineering context loss.

---

# Success looks like this

A developer should be able to ask:

> Why did we migrate away from Redis?

Instead of receiving five unrelated documents...

Project Memory should reconstruct the entire decision journey, including:

- The original problem
- Discussions
- Alternatives
- Final decision
- Implementation
- Outcome

---

# What Project Memory is NOT

Project Memory is NOT:

- A Chat with PDF application
- A generic RAG demo
- A document search engine
- A chatbot for engineering documents

Those may exist internally as building blocks.

They are not the product.

---

# Core Philosophy

Engineering teams don't lose code.

They lose context.

Project Memory exists to preserve that context.
